# vvd.world AI Agent

A locally-running AI agent that automates your [vvd.world](https://vvd.world) worldbuilding account. Type natural language commands and the agent logs in, navigates the site, and performs actions automatically — creating cards, building maps, managing your wiki, and more.

Everything runs on your own machine. No cloud, no subscription, no data sent externally.

---

## Features

- **Browser automation** — logs into vvd.world and controls it via Playwright
- **Natural language commands** — powered by a local Ollama LLM
- **Card creation** — character, location, faction, item, lore, event, magic system, creature, document
- **Map management** — create maps and add interactive pins
- **Wiki configuration** — set title, visibility, and styling
- **Session notes** — create and store campaign session notes
- **D&D web search** — Tavily-powered search for rules, spells, and monsters
- **Campaign knowledge** — reads your lore files and uses them as context
- **Memory** — remembers every action and everything you tell it
- **FastAPI backend** — full REST API with persistent browser session
- **Textual TUI** — hybrid terminal UI with sidebar, command bar, and live logs
- **Intel Arc GPU acceleration** — via IPEX-LLM Ollama build

## Requirements

- Windows 10/11
- Python 3.10, 3.11, or 3.12 (3.13 has known Playwright issues on Windows)
- [Ollama](https://ollama.com) — standard build, or IPEX-LLM build for Intel GPU
- A [vvd.world](https://vvd.world) account
- A [Tavily](https://app.tavily.com) API key (free tier) for D&D web search

## Installation

**1. Clone the repository:**
```bash
git clone https://github.com/yourusername/vvw-agent.git
cd vvw-agent
```

**2. Create and activate a virtual environment:**
```bash
python -m venv .venv
.venv\Scripts\activate
```

**3. Install dependencies:**
```bash
pip install -r requirements.txt
playwright install
```

**4. Copy the example environment file and fill in your details:**
```bash
copy .env.example .env
```

Edit `.env`:
```
VVW_EMAIL=your_email@example.com
VVW_PASSWORD=your_password
OLLAMA_MODEL=vvd-agent
VISION_MODEL=llava
TAVILY_API_KEY=tvly-your_key_here
```

**5. Pull the required Ollama models:**
```bash
ollama pull llama3.1:8b
ollama pull llava
```

**6. Build the custom vvd-agent model:**
```bash
ollama create vvd-agent -f Modelfile
```

## Usage

### One-click launch (recommended)

Double click `start_agent.bat`. This will:
- Kill any existing Ollama processes
- Start Ollama with Intel GPU support (if using IPEX-LLM build)
- Start the FastAPI server
- Start the Textual TUI

### Manual launch

**Terminal 1 — Start the API:**
```bash
uvicorn api.main:app
```

**Terminal 2 — Start the TUI:**
```bash
python tui/app.py
```

### Select your world first

Before creating cards or maps, tell the agent which world to work in. Either type in the TUI:
```
switch to world The Shattered Sky
```

Or call the API directly:
```
POST http://localhost:8000/worlds/select
{"world": "The Shattered Sky"}
```

## Commands

### Cards
```
create a character card for [name]
create a location called [name]
add a faction card for [name]
edit [card name] description to [new text]
delete the card for [name]
link [card a] and [card b] as [relationship]
```

### Maps
```
create a map called [name]
add a pin for [place] on the [map] map
```

### Worlds
```
create a new world called [name]
switch to world [name]
show me the relationship graph
```

### Wiki & Sessions
```
set the wiki title to [title]
create a session note called [title]
```

### Knowledge & AI
```
remember that [lore fact]
what do you know about my campaign?
suggest what I should create next
```

### D&D Questions (web search)
```
how does the silence spell work in D&D 5e?
what monsters live in the underdark?
what are the rules for grappling?
```

### Special
```
help          — show command list
wipe memory   — clear all stored memory
card debug    — test card creation without AI
quit          — exit
```

## Intel GPU Acceleration

Standard Ollama does not support Intel Arc GPUs. To enable GPU acceleration use the [IPEX-LLM Ollama build](https://github.com/intel/ipex-llm/releases):

1. Download the Windows portable zip from the IPEX-LLM releases page
2. Extract to `C:\ipex-ollama\`
3. Use `start_agent.bat` — it automatically starts the GPU-accelerated Ollama server

With GPU acceleration you can expect ~139 tokens/second vs ~13 on CPU.

## Knowledge Files

Drop any `.md` or `.txt` files into the `knowledge/` folder and the agent will automatically read them and use them as context when creating cards and answering questions.

Example:
```
knowledge/

  factions.md    ← faction details
  characters.md  ← character backgrounds
```

## Troubleshooting

**Browser doesn't open**
- Make sure Ollama is running before starting the API
- Try running `start_agent.bat` as Administrator
- Check `http://localhost:8000/health` to see what's connected

**500 errors on card creation**
- Call `POST /worlds/select` first to set the active world
- Check the API terminal for the full Python traceback

**Slow responses**
- Make sure you're using the IPEX-LLM build for Intel GPU
- Check Task Manager → GPU tab — should spike when the model is thinking

**Model not found**
```bash
cd C:\ipex-ollama
ollama.exe create vvd-agent -f C:\path\to\vvw-agent\Modelfile
```

**Web search not working**
- Check `TAVILY_API_KEY` is set correctly in `.env`
- Sign up free at [app.tavily.com](https://app.tavily.com)
