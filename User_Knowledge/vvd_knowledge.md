## What is vvd?

vvd (Vivid) is an all-in-one storytelling and worldbuilding toolkit for writers, dungeon masters, and hobbyists. It lets you organise lore, build interactive maps, write immersive stories, and share your world as a wiki — all in one place. Supports all genres: fantasy, sci-fi, romance, cyberpunk, horror.

**Philosophy:**
- Zero generative AI policy — all tools assist human creativity
- 10% of revenue donated to support real artists
- Content stays private unless you choose to publish
- Full data ownership with export capability

## Cards — Core Data Units

Every piece of lore is stored as a card.

### Card Types

| Type | Use |
|------|-----|
| Character | People, creatures, NPCs, monsters, beings |
| Location | Cities, landmarks, regions, districts, territories |
| Item | Artifacts, gear, resources, magical objects |
| Lore | History, religions, legends, customs, knowledge |
| Ecology | Living organisms, biomes, ecosystems |
| Note | Quick ideas and scratchpad data |
| Custom | Fully user-defined types and sub-types |

### Card Features
- **Title & Aliases** — auto-detected for linking within text
- **Image** — visual representation
- **Properties** — metadata fields (Age, Faction, Habitat, Domain, etc.)
- **Blocks** — modular content sections
- **Guided Templates** — structured prompts to prevent writer's block (e.g. "What was the core reason this city was founded?")

### Card Blocks

| Block | Description |
|-------|-------------|
| Text | Rich text description |
| Photo | Add an image |
| Map Pin | Link card to a map location |
| 5e Stat Block | Full D&D 5e stat block (HP, AC, abilities, actions) |
| 5e Character Sheet | Full player character sheet |
| Family Tree | Visual family relationships |
| Media | Videos and other files |

### Creating a Card (2025 UI)
1. Click **Card** button in editor toolbar
2. Select card type from menu
3. Fill in name in the **New Card** textbox
4. Click **Text** to add description
5. Cards autosave — click **Close** when done

### Organising Cards
- Search bar for finding by name
- Filter by type or document type
- Sort: manually drag, by creation date, or alphabetically
- **Pin**: right-click to pin — moves to top with enlarged image
- **Folders**: named with icons to categorise
- **Parent/child**: drag one card onto another to nest them

## Maps

- Upload any image as a map (Incarnate, Wonderdraft, or custom)
- Drag existing card onto map → creates interactive pin linked to that card
- Right-click map → add empty pin (icon + colour, no card required)
- Updating a card automatically updates its zone data on the map

### Zones
- Click **Zones** button → click map to add vertices
- Close shape by clicking first point (highlighted green when hovering)
- Zones have: label, linked card, colour, opacity, pattern fill
- Points can be added along perimeter to adjust shape

### Layers
- Separate map elements by category (e.g. Cities layer vs Borders layer)
- Toggle visibility with eye icon next to each layer
- Assign zones and pins to specific layers

### Text on Maps
- Click **Text** button → click map
- Edit font, size, scale, layer assignment
- Text spacing can be bent into an arch

### Map Notes
- Background image can be swapped at any time while preserving all pins and zones
- Maps autosave, named in top left corner, found in sidebar
- Custom brushes available for complex geography
- Compatible with Incarnate and Wonderdraft exports

## Canvas View

- Infinite spatial board inspired by Millanote and Obsidian Canvas
- Drag cards, images, and text anywhere
- Draw connections between items visually
- Create mood boards, timelines, and planning layouts
- Useful for session planning and story arcs

## Graph View

- Auto-generated visualisation of all card connections
- Connections created by **mentioning cards in text** OR **adding as properties**
- Inspired by Obsidian Graph View

**Features:**
- Search with magnifying glass — highlights relevant nodes
- Filter by card type and subtype
- Settings: toggle labels, hide isolated nodes, node size slider
- Link distance, strength, repulsion, collision adjustments
- Gravity X and Y — change axis pull for different appearances
- Right-click node to pin it in place
- Save graph configurations to sidebar

## Wiki

- Auto-generated from world content — no manual setup needed
- **Live sync**: edit in wiki → syncs to editor, edit in editor → syncs to wiki
- Custom subdomain: yourworld.vvd.world
- Custom title, description, banner image
- Site styling: theme (colour + font), custom colour palette, custom header/body fonts
- Grid layout on main page: highlight and reorder cards
- Publish with custom domain as public website
- Visitors browse wiki but cannot access your workspace
- Toggle individual cards as wiki-visible or hidden

## Navigation Notes for AI Agent

- Login: click **Continue with Email** before email/password fields appear
- Email field: `input#email` — Password field: `input#password`
- World cards: `button.relative.w-full.text-left.rounded-xl` containing `h3.font-medium`
- After opening world: navigate to `/editor` URL or press `2`
- New card: click Card button, or click center, or right-click sidebar
- Card name: textbox named **New Card**
- Description: click Text button → click empty paragraph → fill `.tiptap.ProseMirror.w-full`
- Cards autosave — click **Close** when done
- Map pin: drag card onto map, or right-click map
- Graph connections: mention cards in text or add as properties
- Wiki: edit directly from wiki tab, changes sync to world editor automatically
